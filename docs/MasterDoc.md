## 1. Project Overview

### Event Name

**DeepTech.AI 2026**

### Theme

**Physical AI**

### Purpose

DeepTech.AI is envisioned as a flagship IEEE Computer Society event focused on Physical AI.

### Mission

DeepTech.AI aims to bridge the gap between digital intelligence and physical systems by bringing together industry leaders, researchers, startups, and students to explore how Artificial Intelligence is transforming robotics, hardware, industrial automation, and cyber-physical systems.

### Vision

Deliver a flagship conference experience that reflects the quality of an international technology summit while remaining aligned with IEEE Computer Society branding.

### Objectives

- Showcase advances in Physical AI.
- Connect academia and industry.
- Promote diversity in AI.
- Encourage startup innovation.

### Audience

Students, researchers, professionals, founders, engineers, executives.

### Success Metrics

- Professional brand perception
- High registration conversion
- Excellent mobile experience
- Future-ready architecture

## 6. Branding Requirements

### Mandatory Logos

The following logos **MUST** appear somewhere appropriate on the site:

- IEEE Computer Society Bangalore Chapter
- GE Healthcare

### Logo Rules (from official IEEE CS Brand Identity & Graphic Style Guide)

- Use official IEEE CS logos only — never redraw, distort, recolor, crop, or recreate.
- The primary logo presents in **black and orange**. It may also present in white and orange, solid black, or solid white.
- The logo must always include the trademark symbol (™) as part of the 1/0 bug element.
- **On black backgrounds:** use the fully white logo.
- **On dark backgrounds** (such as IEEE Blue): use the orange-and-white logo.
- Minimum clear space: **0.3 × logo height** on all sides.
- Never: use the logo as part of a headline or sentence; reconfigure or alter it; place it at an angle; remove elements from it; screen it; or place it on a heavily patterned background or busy photograph. Readability must be ensured in all applications.

### Sub-brand / Chapter Logo Rules

- Sub-brands include the 1/0 bug with the registered trademark symbol plus the IEEE Computer Society name.
- The 1/0 bug (icon) cannot exceed 30% of the entire logo.
- "IEEE COMPUTER SOCIETY" must be uppercase.
- If the sub-brand name is entirely uppercase, "IEEE COMPUTER SOCIETY" should render at 75% of the height of the tallest letter (except Technical Committee logos).
- Chapter logos use the primary logo alongside the full chapter name (e.g., _Bangalore Chapter_), set in italics beneath, with "IEEE COMPUTER SOCIETY" holding greater visual hierarchy — max two lines.
- Logo colors must come from the official IEEE palette; black text is recommended for sub-brand logo consistency.
- Typeface for sub-brand lockups: Montserrat.

### GE Logo

- Must be included per organizer instruction.
- Never redraw, distort, recolor, crop, or recreate.
- Maintain proportional spacing consistent with IEEE logo treatment until GE-specific brand guidelines are provided (see Open Items).

## 7. Official IEEE CS Design System

### Typography

| Role                              | Typeface                                                                   |
| --------------------------------- | -------------------------------------------------------------------------- |
| Primary Headings                  | **Montserrat** (preferred weights: Light, Regular, Medium, Semibold, Bold) |
| Body / Long-form copy             | **Open Sans**                                                              |
| Offline fallback (e.g. MS Office) | **Calibri**                                                                |

- Montserrat is free at: https://fonts.google.com/specimen/Montserrat
- Open Sans is free at: https://fonts.google.com/specimen/Open+Sans
- Both families are used on the Computer Society website: Montserrat for headings and emphasis, Open Sans for long-form copy.

### Primary Color Palette (official — use this as the site's core palette per organizer instruction)

| Name         | Hex       | RGB            | CMYK            | Pantone (Spot)    | RAL  |
| ------------ | --------- | -------------- | --------------- | ----------------- | ---- |
| IEEE Orange  | `#FFA300` | R255 G163 B0   | C0 M41 Y100 K0  | PMS 137 C         | 1003 |
| Process Cyan | `#00B5E2` | R0 G181 B226   | C100 M0 Y0 K0   | PMS Process Cyan  | 5024 |
| IEEE Blue    | `#00629B` | R0 G98 B155    | C100 M35 Y3 K21 | PMS 3015          | 5007 |
| Gray         | `#75787B` | R117 G120 B123 | C30 M22 Y17 K57 | PMS Cool Gray 9C  | 9023 |
| Black        | `#000000` | R0 G0 B0       | C0 M0 Y0 K100   | PMS Process Black | 9017 |
| White        | `#FFFFFF` | R255 G255 B255 | C0 M0 Y0 K0     | —                 | 9016 |

Each primary color also has approved tints at 80% / 60% / 40% / 20% for UI states (hover, disabled, backgrounds, etc).

> **Note on alternate palettes:** The brand guide also defines a "Bright Color Palette" (red, orange, yellow, green, dark green, purple, teal, blue, cyan, white) and a "Dark Color Palette" (darker variants of the same hues) for broader IEEE CS marketing use. **DeepTech.AI should use only the Primary Color Palette above**, per the organizer's explicit instruction to "keep the IEEE CS colour theme" — the bright/dark extended palettes are not part of this project's approved palette unless organizers say otherwise.

- Only official IEEE Computer Society colors should dominate the design.
- Accent colors from the wider IEEE palette may be used sparingly, if at all — default to the six primary colors above.

## 8. Website Vision & Design Philosophy

### The website should feel:

International · Premium · Professional · Engineering-focused · Trustworthy · Modern · Editorial · Research-oriented

### Design Philosophy Keywords

Minimal · Elegant · Spacious · Intelligent · Industrial · Technical · Premium

### Guiding Principle

The design should communicate **confidence rather than visual excess**. Avoid making the site resemble a student event website — this must read as an international technology summit.

### Design Language (from Design System doc)

Premium, minimal, editorial, technical.

### Layout

12-column responsive grid.

### Spacing

8px spacing scale.

### Creative Freedom

AI/contributors may creatively design:

- Hero layouts
- SVG backgrounds
- Motion graphics
- Icons
- Cards
- Timelines
- Responsive layouts
- Section compositions
- Illustration styles
- Scroll animations

**Creativity must never alter factual information.**

## 17. AI & Contributor Operating Rules

Every AI assistant, developer, or contributor working on this project **SHALL**:

1. Preserve brand consistency at all times.
2. Never modify finalized information (Section 2).
3. Never invent event details, speakers, sponsors, or schedules.
4. Use tasteful placeholders where content is pending.
5. Keep all components reusable.
6. Maintain accessibility (WCAG AA) throughout.
7. Follow IEEE Computer Society branding exactly as specified (Sections 6–7).
8. Prioritize quality over visual effects.
9. Design for future expansion (multi-year, multi-edition).
10. Treat this document as the project's constitution — unless explicitly instructed otherwise by the project owner, its contents are authoritative and must not be altered.
11. Keep the registration flow present in code but inactive/closed until organizers explicitly approve opening it.
12. Never fabricate GE-affiliated speakers, quotes, or endorsements — GE's participation as a speaker source is "hoped for," not confirmed (Section 3).
